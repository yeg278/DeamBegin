CPU亲和性调节器使用说明

info 什么是CPU亲和性

CPU亲和性（CPU Affinity）是指将特定进程或线程绑定到指定的CPU核心上运行的技术。通过合理设置CPU亲和性，可以：

  • 降低功耗，延长电池续航
  • 提高性能，减少卡顿
  • 减轻发热，提升用户体验
  • 实现更精细化的性能调优

settings 基本概念

在设置CPU亲和性时，需要了解以下概念：

  • 主进程：应用的主要进程，通常与应用包名相同
  • 子进程：应用创建的额外进程，通常以冒号开头，如 :push
  • 线程：进程内部的执行单元，如 RenderThread 渲染线程
  • CPU核心：现代设备通常有多个CPU核心，编号从0开始
  • 大小核架构：如在骁龙处理器中，小核(0-3)更节能，大核(4-7)性能更高

menu 功能菜单

右下角的悬浮按钮菜单提供以下功能：

  • brightness_6 切换主题：在深色模式和浅色模式之间切换
  • help_outline 使用说明：打开本使用指南
  • refresh 刷新：重新加载页面数据，当显示异常或需要重置状态时使用
  • save 保存：保存当前的所有配置设置
  • add 添加应用：添加新的应用规则

add_circle 添加绑定规则

添加CPU亲和性绑定规则的步骤：

  1. 点击右下角的add按钮，打开添加规则窗口
  2. 输入应用名称（可选，便于识别）和包名（必填）
  3. 添加进程/线程规则：
    • 输入进程/线程名（子进程需以冒号开头，如 :push）
    • 设置绑定的CPU核心范围
    • 可点击"添加进程/线程"按钮添加多个规则
  4. 点击"添加"按钮保存规则

lightbulb 提示：对于已有的应用，可以点击应用卡片上的add按钮快速添加新的规则。

edit 管理绑定规则

点击应用卡片可展开查看所有已设置的绑定规则，您可以：

  • 编辑规则：点击规则右侧的edit按钮，修改包名、进程/线程类型和CPU核心设置
  • 修改核心：直接编辑规则右侧的核心输入框，修改后自动保存
  • 删除规则：点击规则右侧的delete按钮删除单条规则
  • 删除应用：点击应用卡片上的delete按钮删除该应用的所有规则
  • 编辑应用名：点击应用卡片上的drive_file_rename_outline按钮修改应用名称

save 保存配置

当您完成规则设置后，请点击右下角的save按钮保存您的配置。未保存的更改在页面刷新后将丢失。

settings_ethernet CPU核心设置格式

CPU核心范围支持以下格式：

格式 示例 说明
单个核心 0 仅使用第1个核心（从0开始计数）
连续范围 0-3 使用第1到第4个核心
多个范围 0-3,6-7 使用第1到第4个核心和第7到第8个核心
分散核心 0,2,4,6 使用第1,3,5,7个核心

search 查找应用

使用页面顶部的搜索框可以快速查找已添加的应用。搜索支持：

  • 应用名称
  • 包名
  • 进程/线程名

搜索时会自动高亮匹配的结果并展开相关卡片。
